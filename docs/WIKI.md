## Обзор

Flamix - это система централизованного управления файрволом с клиент-серверной архитектурой. Она позволяет управлять правилами файрвола на множестве клиентских машин из единого центра.

### Основные возможности

- **Централизованное управление**: Управляйте правилами файрвола для всех клиентов из одного места
- **Автоматическая синхронизация**: Правила автоматически синхронизируются с клиентами
- **Мониторинг изменений**: Обнаружение и авторизация ручных изменений правил
- **Аналитика**: Сбор статистики блокировок и попыток подключения
- **Безопасность**: Шифрование, TLS, защита от replay-атак

### Протокол связи

Flamix использует собственный протокол поверх TLS:

1. **Установление соединения**: TLS handshake с mutual authentication
2. **Обмен ключами**: Diffie-Hellman для генерации сессионного ключа
3. **Аутентификация**: Проверка сертификатов
4. **Обмен сообщениями**: Зашифрованные сообщения с HMAC

### Типы сообщений

- `AUTH` - Аутентификация клиента
- `SYNC` - Синхронизация правил
- `RULE_UPDATE` - Обновление правила
- `ANALYTICS` - Отправка аналитики
- `HEARTBEAT` - Проверка соединения
- `KEY_ROTATION` - Ротация ключей
- `DH_KEY_EXCHANGE` - Обмен ключами Diffie-Hellman

## Установка и настройка

### Установка сервера

1. Клонируйте репозиторий и перейдите в ветку `server`:
   ```bash
   git clone <repository-url>
   git checkout server
   cd server
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Запустите сервер:
   ```bash
   python run.py
   ```

### Установка клиента

1. Клонируйте репозиторий и перейдите в ветку `client`:
   ```bash
   git clone <repository-url>
   git checkout client
   cd client
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Получите сертификат от сервера (см. [Подключение клиентов](#подключение-клиентов))

4. Запустите клиент:
   ```bash
   python run.py --client-id my-client --server-host <server-ip>
   ```

### Установка GUI приложения

1. Клонируйте репозиторий и перейдите в ветку `app`:

## Использование

### Подключение клиентов

#### Через веб-интерфейс

1. Откройте веб-интерфейс: `http://127.0.0.1:8080`
2. Перейдите в раздел "Clients"
3. Нажмите "Add Client"
4. Введите Client ID и имя
5. Скачайте сертификат клиента
6. Передайте сертификат на клиентскую машину

#### Через API

```bash
# Создать клиента
curl -X POST http://127.0.0.1:8080/api/clients \
  -H "Content-Type: application/json" \
  -d '{"client_id": "my-client", "name": "My Client"}'

# Получить сертификат
curl http://127.0.0.1:8080/api/clients/my-client/certificate
```

### Управление правилами

#### Через веб-интерфейс

1. Откройте веб-интерфейс
2. Перейдите в раздел "Rules"
3. Выберите клиента
4. Нажмите "Add Rule" для создания нового правила
5. Заполните форму и сохраните

#### Через API

```bash
# Получить правила клиента
curl http://127.0.0.1:8080/api/clients/my-client/rules

# Добавить правило
curl -X POST http://127.0.0.1:8080/api/clients/my-client/rules \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Block Malicious IPs",
    "action": "block",
    "direction": "inbound",
    "protocol": "TCP",
    "targets": {
      "ips": ["192.168.1.100"]
    }
  }'
```

### Мониторинг и аналитика

#### Просмотр аналитики

1. Откройте веб-интерфейс
2. Перейдите в раздел "Analytics"
3. Выберите клиента или "all" для всех клиентов
4. Просмотрите статистику блокировок

#### Запросы на изменение

1. Откройте веб-интерфейс
2. Перейдите в раздел "Change Requests"
3. Просмотрите запросы от клиентов
4. Одобрите или отклоните запросы

## Безопасность

### Шифрование базы данных

База данных сервера шифруется с помощью SQLCipher. Ключи автоматически ротируются каждые 24 часа.

### TLS соединения

Все соединения между клиентом и сервером защищены TLS с mutual authentication:
- Сервер проверяет сертификат клиента
- Клиент проверяет сертификат сервера

### Защита от атак

- **Replay-атаки**: Использование nonce и timestamps
- **Man-in-the-middle**: Mutual TLS
- **Подмена сообщений**: HMAC для проверки целостности
- **Перехват ключей**: Diffie-Hellman для безопасного обмена ключами

### Рекомендации по безопасности

1. Храните сертификаты в безопасном месте
2. Используйте сильные пароли для ключей базы данных
3. Регулярно обновляйте сертификаты
4. Мониторьте логи на подозрительную активность
5. Используйте файрвол для ограничения доступа к серверу

## Troubleshooting

### Проблемы с подключением

**Клиент не может подключиться к серверу**

1. Проверьте, что сервер запущен
2. Проверьте правильность `--server-host` и `--server-port`
3. Убедитесь, что сертификаты находятся в правильной директории
4. Проверьте, что файрвол не блокирует соединение

**Ошибка сертификата**

1. Убедитесь, что сертификаты не истекли
2. Проверьте, что сертификат клиента соответствует Client ID
3. Убедитесь, что CA сертификат правильный

### Проблемы с правилами

**Правила не применяются**

1. Убедитесь, что клиент запущен с правами администратора
2. Проверьте, что установлены необходимые плагины
3. Проверьте логи на наличие ошибок конвертации

**Правила не синхронизируются**

1. Проверьте соединение между клиентом и сервером
2. Проверьте логи на наличие ошибок
3. Убедитесь, что интервал синхронизации настроен правильно

### Проблемы с базой данных

**Ошибка открытия базы данных**

1. Проверьте, что ключ базы данных правильный
2. Убедитесь, что файл базы данных не поврежден
3. Попробуйте восстановить из резервной копии

**База данных заблокирована**

1. Убедитесь, что только один процесс использует базу данных
2. Проверьте, что нет зависших соединений
3. Перезапустите сервер
