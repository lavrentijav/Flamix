# Упрощенная установка клиента

## Создание клиента через GUI

1. Запустите GUI приложение:
   ```bash
   cd app
   python run.py
   ```

2. Подключитесь к серверу (введите URL и нажмите "Connect")

3. Перейдите на вкладку "Clients"

4. Нажмите "Add Client"

5. Введите:
   - **Client ID** - уникальный идентификатор клиента (например, `workstation-01`)
   - **Client Name** - отображаемое имя (опционально)

6. Нажмите "Create & Download"

7. ZIP архив будет сохранен в директорию `Downloads/`

## Установка клиента

### Шаг 1: Распаковка

Распакуйте ZIP архив в любую директорию:

```bash
unzip flamix-client-workstation-01.zip
cd flamix-client-workstation-01
```

### Шаг 2: Установка зависимостей

```bash
pip install -r requirements.txt
```

### Шаг 3: Запуск

```bash
python run.py
```

**Всё готово!** Клиент автоматически:
- Подключится к серверу
- Синхронизирует правила
- Начнет мониторинг изменений

## Что входит в ZIP архив

```
flamix-client-{client_id}/
├── config.json          # Конфигурация (IP сервера, порты, интервалы)
├── certs/              # Сертификаты
│   ├── ca.crt          # Сертификат CA
│   ├── client.crt      # Сертификат клиента
│   └── client.key      # Приватный ключ клиента
├── flamix/             # Модули клиента (если включены)
│   ├── client/
│   └── common/
├── run.py              # Скрипт запуска
├── requirements.txt    # Зависимости Python
└── README.md           # Инструкция
```

## Настройка конфигурации

Отредактируйте `config.json` перед запуском, если нужно изменить настройки:

```json
{
  "client_id": "workstation-01",
  "server_host": "192.168.1.100",
  "server_port": 8443,
  "cert_dir": "certs",
  "sync_interval": 30,
  "monitor_interval": 10,
  "analytics_enabled": false,
  "analytics_interval": 60
}
```

## Если модули не включены в архив

Если в ZIP архиве нет директории `flamix/`, скопируйте её из ветки `client/`:

```bash
# Из корня проекта
cp -r client/flamix flamix-client-workstation-01/
```

## Создание клиента через API

```bash
# Создать клиента и получить ZIP
curl -X POST http://127.0.0.1:8080/api/clients \
  -H "Content-Type: application/json" \
  -d '{"client_id": "my-client", "name": "My Client"}' \
  --output client-package.zip

# Получить пакет для существующего клиента
curl http://127.0.0.1:8080/api/clients/my-client/package \
  --output client-package.zip
```

## Автоматическое определение IP сервера

Сервер автоматически определяет свой IP адрес для включения в конфиг клиента:
- Если сервер слушает на `0.0.0.0`, определяется реальный IP адрес
- IP адрес включается в `config.json` в поле `server_host`

## Безопасность

- **Приватный ключ клиента** находится в `certs/client.key`
- **Не передавайте ZIP архив третьим лицам!**
- Храните архив в безопасном месте
- При компрометации ключа создайте нового клиента

## Устранение неполадок

### Клиент не может подключиться

1. Проверьте, что сервер запущен
2. Проверьте `server_host` в `config.json` - должен быть IP сервера, а не `localhost`
3. Убедитесь, что порт `server_port` (8443) не заблокирован файрволом
4. Проверьте, что сертификаты находятся в `certs/`

### Ошибка импорта модулей

1. Убедитесь, что установлены зависимости: `pip install -r requirements.txt`
2. Если модули не в архиве, скопируйте `flamix/` из ветки `client/`
3. Проверьте, что Python версии 3.8+

### Ошибка сертификата

1. Убедитесь, что сертификаты не повреждены
2. Проверьте, что `client_id` в конфиге совпадает с ID на сервере
3. Пересоздайте клиента, если сертификат истек
