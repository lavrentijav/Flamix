# Установка сертификатов для Flamix

## Автоматическая генерация

При первом запуске сервера сертификаты генерируются автоматически в директории `certs/`:
- `ca.crt` - корневой сертификат CA
- `ca.key` - приватный ключ CA
- `server.crt` - сертификат сервера
- `server.key` - приватный ключ сервера

## Использование для веб-интерфейса

### Вариант 1: HTTP (без SSL) - для локального использования

Если сертификаты отсутствуют, веб-интерфейс автоматически запустится по HTTP:
```
http://127.0.0.1:8080
```

**Важно:** Это безопасно только для локального использования!

### Вариант 2: HTTPS с самоподписанными сертификатами

1. Запустите сервер один раз - сертификаты будут созданы автоматически
2. Откройте браузер и перейдите на `https://127.0.0.1:8080`
3. Браузер покажет предупреждение о самоподписанном сертификате
4. Нажмите "Дополнительно" → "Перейти на сайт" (или "Advanced" → "Proceed")

### Вариант 3: Использование собственных сертификатов

Если у вас есть собственные сертификаты:

1. Поместите файлы в директорию `certs/`:
   - `server.crt` - сертификат сервера
   - `server.key` - приватный ключ сервера

2. Перезапустите сервер

3. Веб-интерфейс автоматически использует эти сертификаты

## Для клиентов

Клиенты должны иметь:
- `ca.crt` - корневой сертификат CA (для проверки сервера)
- `client.crt` - клиентский сертификат (если требуется mutual TLS)
- `client.key` - приватный ключ клиента

Эти файлы должны быть в директории `certs/` на клиенте.

## Проверка сертификатов

Чтобы проверить, что сертификаты созданы:

```bash
# Windows
dir certs

# Linux/Mac
ls -la certs/
```

Должны быть файлы:
- `ca.crt`
- `ca.key`
- `server.crt`
- `server.key`

## Удаление и пересоздание

Если нужно пересоздать сертификаты:

1. Остановите сервер
2. Удалите директорию `certs/` или файлы в ней
3. Запустите сервер снова - сертификаты будут созданы заново

## Безопасность

⚠️ **Важно:**
- Самоподписанные сертификаты подходят только для разработки и тестирования
- Для продакшена используйте сертификаты от доверенного CA (Let's Encrypt и т.д.)
- Никогда не коммитьте приватные ключи в репозиторий!
